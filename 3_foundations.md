# 训练

## 3.1 损失函数

方程 1.1 中的均方误差示例是预测连续值的标准损失函数。

对于密度建模，标准损失是数据的似然。如果 f(x;w) 被解释为归一化的对数概率或对数密度，则损失是其值在训练样本上求和取负，这对应于数据集的似然。

**交叉熵**
对于分类，通常的策略是模型的输出是一个向量，每个类别 y 对应一个分量$f(x;w)_y$，解释为非归一化概率的对数，即 logit。

设 X 为输入信号，Y 为待预测的类别，我们可以从 f 计算后验概率的估计：


$$
\hat{P}(Y = y \mid X = x) = \frac{\exp f(x;w)_y}{\sum_z \exp f(x;w)_z}
$$


这个表达式通常称为 logits 的 softmax，或更恰当地称为 softargmax。

为了与此解释一致，应训练模型以最大化真实类别的概率，从而最小化交叉熵，表达式为：


$$
\mathcal{L}_{ce}(w) = -\frac{1}{N} \sum_{n=1}^{N} log\hat{P}(Y = y_n | X = x_n)
$$


$$
= \frac{1}{N}\sum_{n=1}^{N}-log\frac{exp f(x_n;w)_{y_n}}{\sum_z exp f(x_n;w)_z}
$$


**对比损失**
在某些设置中，即使要预测的值是连续的，监督也以排序约束的形式出现。这种情况的典型领域是度量学习，其目标是学习样本之间的距离度量，使得来自某个语义类别的样本$x_a$ 与同类的任何样本$x_b$ 比与来自另一类的任何样本 $x_c$ 更接近。例如，$x_a$ 和$x_b$ 可以是某个人的两张照片，而$x_c$ 是其他人的照片。

此类情况的标准方法是最小化对比损失，例如，对于三元组$(x_a, x_b, x_c)$（满足$y_a = y_b \neq y_c$）求和：


$$
max(0, 1 - f(x_a, x_c; w) + f(x_a, x_b; w))
$$


除非$f(x_a, x_c; w) \geq 1 + f(x_a, x_b; w)$，否则该值严格为正。

**设计损失函数**
通常，训练期间最小化的损失并非最终要优化的实际量，而是更容易找到最佳模型参数的代理。例如，交叉熵是分类的标准损失，即使实际的性能度量是分类错误率，因为后者没有信息梯度（这是关键要求，我们将在 §3.3 中看到）。

也可以向损失中添加依赖于模型可训练参数本身的项，以偏好某些配置。例如，权重衰减正则化包括向损失添加一个与参数平方和成比例的项。这可以解释为对参数具有高斯贝叶斯先验，偏好较小的值，从而减少数据的影响。这会降低训练集上的性能，但减少训练性能与新的未见数据性能之间的差距。

## 3.2 自回归模型

一个重要的方法类别，特别是用于处理自然语言处理和计算机视觉中的离散序列，是自回归模型。

**概率的链式法则**
此类模型利用了概率论中的链式法则：


$$
P(X_1 = x_1, X_2 = x_2, ..., X_\tau = x_\tau) = P(X_1 = x_1) \times P(X_2 = x_2 | X_1 = x_1) \times ... \times P(X_\tau = x_\tau | X_1 = x_1, ..., X_{\tau-1} = x_{\tau-1})
$$


尽管这种分解对任何类型的随机序列都有效，但当感兴趣信号是来自有限词汇表 {1,...K} 的令牌序列时，它特别高效。按照约定，额外令牌$\varnothing$ 表示“未知”量，我们可以将事件${X_1 = x_1, ..., X_t = x_t}$ 表示为向量$(x_1, ..., x_t, \varnothing, ..., \varnothing)$。

然后，一个模型$f: {\varnothing,1,...,K}^\tau \to \mathbb{R}^{K}$，给定这样的输入，计算一个包含 K 个 logits 的向量 l_t，对应$P̂(X_t | X_1 = x_1, ..., X_{t-1} = x_{t-1})$，允许在给定先前令牌的情况下采样一个令牌。

链式法则确保通过逐个采样$\tau$ 个令牌 ${x_t}$（在给定先前采样的$x_1, ..., x_{t-1}$ 的情况下），我们得到一个遵循联合分布的序列。这是一个自回归生成模型。

训练这样的模型可以通过最小化训练序列和时间步上的交叉熵损失之和来完成：$L_ce(f(x_1, ..., x_{t-1}, \varnothing, ..., \varnothing; w), x_t)$，这在形式上等价于最大化真实 ${x_t}$ 的似然。

传统上监测的值不是交叉熵本身，而是困惑度，定义为交叉熵的指数。它对应于具有相同熵的均匀分布的值数量，通常更具可解释性。

**因果模型**
我们刚刚描述的训练过程需要为每个 t 提供不同的输入，并且为 t < t′ 所做的大量计算会为 t′ 重复。这非常低效，因为$\tau$ 通常为数百或数千。

解决此问题的标准策略是设计一个模型 f，一次性预测所有 logits 向量$l_1, ..., l_\tau$，即：$f: {1,...,K}^\tau \to \mathbb{R}^{\tau\timesK}$，但具有这样的计算结构：为 ${x_t}$ 计算的 logits l_t 仅依赖于输入值$x_1, ..., x_{t-1}$。

这种模型称为因果模型，因为在时间序列的情况下，它对应于不让未来影响过去，如图 3.1 所示。结果是每个位置的输出将是如果输入仅在该位置之前可用时获得的输出。在训练期间，它允许计算完整序列的输出，并最大化该同一序列所有令牌的预测概率，这再次归结为最小化每令牌交叉熵之和。

请注意，为简单起见，我们将 f 定义为操作固定长度$\tau$ 的序列。然而，实践中使用的模型，例如我们将在 §5.3 中看到的 transformers，能够处理任意长度的序列。

**分词器**
处理自然语言时的一个重要技术细节是令牌的表示可以有多种方式，从最细粒度的单个符号到整个单词。与令牌表示之间的转换由称为分词器的单独算法执行。

标准方法是字节对编码（BPE）[Sennrich et al., 2015]，它通过分层合并字符组来构建令牌，尝试获得表示各种长度但频率相似的单词片段的令牌，将令牌分配给长频繁片段以及稀有单个符号。

## 3.3 梯度下降

除了像我们在 §1.2 中看到的线性回归这样的特定情况外，最优参数 w* 没有闭式表达式。在一般情况下，最小化函数的首选工具是梯度下降。它首先用随机$w_0$ 初始化参数，然后通过迭代梯度步来改进此估计，每一步包括计算损失相对于参数的梯度，并减去其一小部分：


$$
w_{n+1} = w_n - \eta \nabla\mathcal{L}|_w(w_n)。 \tag{3.1}
$$


此过程对应于将当前估计值沿局部最大程度减小$\mathcal{L}(w)$ 的方向移动一点，如图 3.2 所示。

**学习率**
超参数$\eta$ 称为学习率。它是一个正值，调节最小化的速度，必须谨慎选择。如果太小，优化最多会很慢，并且可能早期陷入局部最小值。如果太大，优化可能会在良好最小值附近反弹，永远无法下降进去。正如我们将在 §3.6 中看到的，它可能依赖于迭代次数 n。

**随机梯度下降**
实践中使用的所有损失都可以表示为每个小组样本或每个样本的损失的平均值，例如：


$$
\mathcal{L}(w) = (1/N) \sum_{n=1}^{N} \ell_n(w),
$$


其中$\ell_n(w) = L(f(x_n; w), y_n)$ 对于某个 L，梯度则为：


$$
\nabla\mathcal{L}|_w(w) = (1/N) \sum_{n=1}^{N} \nabla\ell_n|_w(w)。 \tag{3.2}
$$


由此产生的梯度下降将精确计算方程 3.2 中的和，这通常计算量很大，然后根据方程 3.1 更新参数。然而，在合理的可交换性假设下（例如，如果样本已正确洗牌），方程 3.2 的任何部分和都是完整和的无偏估计，尽管有噪声。因此，从部分和更新参数对应于在相同计算预算下进行更多梯度步，但梯度估计更嘈杂。由于数据的冗余性，这恰好是一种更高效的策略。

我们在 §2.1 中看到，处理一个足够小以适合计算设备内存的样本批次通常与处理单个样本一样快。因此，标准方法是将完整集$\mathcal{D}$ 拆分为批次，并根据从每个批次计算的梯度估计更新参数。这称为小批量随机梯度下降，或简称为随机梯度下降（SGD）。

需要注意的是，这个过程极其渐进，小批次和梯度步的数量通常为数百万。

与许多算法一样，直觉在高维中失效，尽管可能认为这个过程很容易陷入局部最小值，但实际上，由于参数数量、模型设计以及数据的随机性，其效率远超预期。

这种标准策略有很多变体已被提出。最流行的是 Adam [Kingma and Ba, 2014]，它保持每个梯度分量的均值和方差的运行估计，并自动归一化它们，避免缩放问题以及模型不同部分的不同训练速度。

## 3.4 反向传播

使用梯度下降需要一种技术手段来计算$\nabla\ell|_w(w)$，其中$\ell = L(f(x; w); y)$。鉴于 f 和 L 都是标准张量操作的组合，对于任何数学表达式，微分学中的链式法则允许我们获得其表达式。

为了简化符号，我们将不指定梯度在哪个点计算，因为上下文很清楚。

**前向传播和反向传播**
考虑映射组合的简单情况：


$$
f = f^{(D)} \circ f^{(D-1)} \circ \cdots \circ f^{(1)}
$$


f(x; w) 的输出可以通过从$x^{(0)} = x$ 开始并迭代应用来计算：

$x^{(d)} = f^{(d)}(x^{(d-1)}; w_d)$，其中$x^{(D)}$ 为最终值。

这些中间结果$x^{(d)}$ 的单个标量值传统上称为激活（参考神经元激活），值 D 是模型的深度，单个映射$f^{(d)}$ 称为层（我们将在 §4.1 中看到），它们的顺序评估是前向传播（见图 3.3，顶部）。

相反，损失相对于$f^{(d-1)}$ 输出$x^{(d-1)}$ 的梯度$\nabla\ell \mid_{x^{(d-1)}}$ 是损失相对于$f^{(d)}$ 输出$\nabla\ell \mid_{x^{(d)}}$ 的梯度乘以$f^{(d-1)}$ 相对于其变量 x 的雅可比矩阵$J_{f^{(d-1)} \mid x}$ 的乘积。因此，可以递归向后计算相对于所有$f^{(d)}$ 输出的梯度，从$\nabla\ell \mid_{x^{(D)}} = \nabla L \mid_{x}$ 开始。

而我们训练感兴趣的梯度，即$\nabla\ell \mid_{w_d}$，是损失相对于$f^{(d)}$ 输出的梯度乘以$f^{(d)}$ 相对于参数的雅可比矩阵$J_{f^{(d)} \mid w}$ 的乘积。

这种相对于中间激活的梯度的迭代计算，结合相对于层参数的梯度计算，是反向传播（见图 3.3，底部）。此计算与梯度下降过程的结合称为反向传播。

实际上，前向传播和反向传播的实现细节对程序员是隐藏的。深度学习框架能够自动构建计算梯度的操作序列。

一个特别方便的算法是 Autograd [Baydin et al., 2015]，它跟踪张量操作并即时构建梯度算子的组合。因此，一段操作张量的命令式编程可以自动计算任何量相对于任何其他量的梯度。

**资源使用**
关于计算成本，正如我们将看到的，大部分计算用于线性操作，每个操作需要一次矩阵乘积用于前向传播，两次用于反向传播中乘以雅可比矩阵，使得后者成本大约是前者的两倍。

推理期间的内存需求大致等于最苛刻的单个层的需求。然而，对于训练，反向传播需要保留前向传播期间计算的激活以计算雅可比矩阵，这导致内存使用与模型深度成比例增长。存在通过依赖可逆层 [Gomez et al., 2017] 或使用检查点技术来交换内存使用和计算的方法，检查点技术包括仅存储某些层的激活，并在反向传播期间通过部分前向传播即时重新计算其他层的激活 [Chen et al., 2016]。

**梯度消失**
训练大型网络时的一个关键历史问题是，当梯度通过算子反向传播时，它可能被乘以一个因子，因此在穿过许多层时呈指数级减小或增加。防止其爆炸的标准方法是梯度范数裁剪，包括重新缩放梯度，如果其范数高于固定阈值，则将其范数设置为该阈值 [Pascanu et al., 2013]。

当梯度呈指数级减小时，这称为梯度消失，它可能使训练无法进行，或者在其较温和的形式中，导致模型的不同部分以不同速度更新，降低它们的共同适应 [Glorot and Bengio, 2010]。

正如我们将在第 4 章中看到的，已经开发了多种技术来防止这种情况发生，反映了一个对深度学习成功至关重要的视角转变：不再试图改进通用优化方法，而是转向设计模型本身以使其可优化。

## 3.5 深度的价值

正如“深度学习”一词所示，有用的模型通常是长系列映射的组合。用梯度下降训练它们会导致映射的复杂共同适应，尽管这个过程是渐进和局部的。

我们可以用一个简单的$\mathbb{R}^{2} \to \mathbb{R}^{2}$ 模型来说明此行为，该模型结合了八个层，每个层将其输入乘以一个$2 \times 2$ 矩阵并按分量应用 Tanh，最后是一个线性分类器。这是标准多层感知机的简化版本，我们将在 §5.1 中看到。

如果我们在一个玩具二元分类任务上使用 SGD 和交叉熵训练此模型（图 3.4，左上），矩阵会共同适应以变形空间，直到分类正确，这意味着在最后的仿射操作之前数据已变得线性可分（图 3.4，右下）。

这样的例子让我们一瞥深度模型可以实现什么；然而，由于待处理信号和内部表示的维度都很低，它部分具有误导性。这里一切都保持在二维以便可视化，而真实模型利用了高维表示，这尤其通过提供许多自由度来促进优化。

过去二十年来积累的经验证据表明，跨应用领域的最先进性能需要具有数十层的模型，例如残差网络（见 §5.2）或 Transformers（见 §5.3）。理论结果表明，对于固定的计算预算或参数数量，增加深度会导致所得映射的复杂度更高 [Telgarsky, 2016]。

## 3.6 训练协议

训练深度网络需要定义一个协议，以充分利用计算和数据，并确保在新数据上性能良好。

正如我们在 §1.3 中看到的，训练样本上的性能可能具有误导性，因此在最简单的设置中至少需要两组样本：一组是训练集，用于优化模型参数，另一组是测试集，用于评估训练后模型的性能。

此外，通常需要调整超参数，特别是与模型架构、学习率和损失中正则化项相关的超参数。在这种情况下，需要一个与训练集和测试集都不相交的验证集来评估最佳配置。

完整训练通常分解为周期，每个周期对应遍历所有训练样本一次。损失的通常动态是，只要优化运行，训练损失就会减少，而验证损失可能在经过一定数量的周期后达到最小值，然后开始增加，反映了过拟合机制，如 §1.3 中介绍和图 3.5 所示。

矛盾的是，尽管由于容量大，大型模型本应遭受严重的过拟合，但它们通常随着训练进展而继续改进。这可能是因为当训练集上的性能接近完美时，模型的归纳偏置成为优化的主要驱动力 [Belkin et al., 2018]。

**学习率调度**
一个重要的设计选择是训练期间的学习率调度，即指定梯度下降每次迭代的学习率值。一般策略是学习率最初应较大以避免优化早期陷入不良局部最小值，然后应变小以使优化的参数值不会在损失景观的狭窄山谷中反弹并达到良好最小值。

非常大的模型的训练可能在数千个强大 GPU 上花费数月，财务成本高达数百万美元。在这个规模上，训练可能涉及许多手动干预，特别是由损失演变动态所指导。

**微调**
通常有益于将已训练的模型适应新任务，称为下游任务。这可能是因为原始任务的数据量丰富，而下游任务的数据有限，并且两个任务共享足够的相似性，以至于为第一个任务学习的统计结构为第二个任务提供了良好的归纳偏置。也可能是为了通过重用现有模型中编码的模式来限制训练成本。

将预训练模型适应特定任务通过微调实现，微调是下游任务的标准训练过程，但从预训练模型开始，而不是使用随机初始化。

这是大多数计算机视觉应用的主要策略，这些应用通常使用在 ImageNet [Deng et al., 2009] 上预训练进行分类的模型（见 §6.3 和 §6.4），这也是纯生成式预训练大型语言模型被重新用作类似助手的模型、能够产生交互式对话的方式（见 §7.1）。

我们将在第 8 章中回到应对有限推理资源和微调的技术。

## 3.7 规模的好处

有大量的经验结果表明，性能（例如通过测试数据上的损失估计）随着数据量的增加而提高，遵循显著的缩放定律，只要模型尺寸相应增加 [Kaplan et al., 2020]（见图 3.6）。

在数十亿样本体系中受益于这些缩放定律部分归功于深度模型的结构，正如我们将看到的，通过增加层数或特征维度可以任意扩展。但也归功于它们实现的分布式计算特性，以及随机梯度下降，它一次只需要一部分数据，并且可以处理大小比计算设备内存大几个数量级的数据集。这导致模型呈指数增长，如图 3.7 所示。

典型的视觉模型有 10-100 百万个可训练参数，训练需要$10^{18} - 10^{19} FLOPs [He et al., 2015; Sevilla et al., 2022]$。语言模型有从 1 亿到数千亿个可训练参数，训练需要$10^{20} - 10^{23} FLOPs [Devlin et al., 2018; Brown et al., 2020; Chowdhery et al., 2022; Sevilla et al., 2022]$。后者需要配备多个高端 GPU 的机器。

训练这些大型模型不可能使用具有详细真实值的数据集，这些数据集制作成本高，只能规模适中。相反，它是通过结合互联网上可用数据并经过最少（如果有）筛选自动生成的数据集来完成的。这些集合可能结合多种模态，例如来自网页的文本和图像，或来自视频的声音和图像，可用于大规模监督训练。

截至 2024 年，最强大的模型是所谓的大型语言模型（LLMs），我们将在 §5.3 和 §7.1 中看到，它们是在极大的文本数据集上训练的（见表 3.1）。
